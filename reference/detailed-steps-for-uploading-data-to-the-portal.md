## 1. Create dataset on the open data portal & add initial metadata

| Step                                                                                                                                                                                                                                                            | Screenshot                                   |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| [Login to the open data portal](https://data.ca.gov/user/login)                                                                                                                                                                                                 | ![](../.gitbook/assets/login.png)            |
| Click on the **My Datasets** tab                                                                                                                                                                                                                                |  ![](../.gitbook/assets/my\_datasets.png)    |
| Click the button labeled **Add Dataset**                                                                                                                                                                                                                        | ![](../.gitbook/assets/add\_dataset.png)     |
| Enter metadata by copying from the Metadata Template to the relevant fields. [Step 3 in this handbook covers the creation of metadata](../3.-create-metadata-and-data-dictionary.md). Fields are ordered in the template the same as they are in the interface. | ![](../.gitbook/assets/metadata\_fields.png) |
| Ensure the field License is entered as Other (Public Domain)                                                                                                                                                                                                    | ![](../.gitbook/assets/license.png)          |
| Ensure the **Visibility** is set to **Private.** This is the default.                                                                                                                                                                                           | ![](../.gitbook/assets/visibility.png)       |
| Click the button labeled **Next: Add Data**                                                                                                                                                                                                                     | ![](../.gitbook/assets/add\_data.png)        |

## 2. Upload data resources

| Step                                                                                                                                                                                                                                                                  | Screenshot                                               |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| After clicking Next Add Data in the previous step, you will see an interface to add files by uploading or linking                                                                                                                                                     | ![](../.gitbook/assets/add\_data\_interface.png)         |
| Click the button labeled **Upload**                                                                                                                                                                                                                                   | ![](../.gitbook/assets/upload\_button.png)               |
| Select your data file and click Open. Data files must be in an open format like CSV.                                                                                                                                                                                  | ![](../.gitbook/assets/select\_file.png)                 |
| Add a **Title** and **Description.** See guidance on [writing titles](../3.-create-metadata-and-data-dictionary.md#best-practices-dataset-title-content) and [descriptions](../3.-create-metadata-and-data-dictionary.md#best-practices-dataset-description-content). | ![](../.gitbook/assets/resource\_title\_description.png) |
| Do not enter anything in **Format**. This will be detected by the system.                                                                                                                                                                                             | ![](../.gitbook/assets/format.png)                       |
| <p>If you have another data file to upload, click the button labeled <strong>Save &#x26; add another</strong>. <br><br>Repeat the steps starting at the top of this section.</p>                                                                                      | ![](../.gitbook/assets/save\_add\_another.png)           |
| If you want to add more non-data resources like documentation, click the button labeled **Save & add another** and skip to the next section where you'll continue adding non-data files.                                                                              | ![](<../.gitbook/assets/save\_add\_another (1).png>)     |
| If you are done adding data files and have no other files to add in the next section, click the button labeled **Finish**                                                                                                                                             | ![](../.gitbook/assets/finish.png)                       |

## 3. If applicable, upload non-data related resources (guides, documentation, etc)

| Step                                                                                                                                                                                 | Screenshot                                               |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------- |
| If you do not have additional non-data resources to add, you can skip this section                                                                                                   |                                                          |
| Click the button labeled **Upload**                                                                                                                                                  | ![](../.gitbook/assets/upload\_button.png)               |
| Select your non-data file and click Open. If you are providing additional reference documentation, PDF is the best format to provide this in.                                        | ![](../.gitbook/assets/upload\_non-data.png)             |
| Add a **Title** and **Description**                                                                                                                                                  | ![](../.gitbook/assets/non-data\_title\_description.png) |
| Do not enter anything in **Format**. This will be detected by the system.                                                                                                            | ![](../.gitbook/assets/format.png)                       |
| <p>If you have another non-data file to upload, click the button labeled <strong>Save &#x26; add another</strong>. <br><br>Repeat the steps starting at the top of this section.</p> | ![](<../.gitbook/assets/save\_add\_another (1).png>)     |
| If you are done adding files, click the button labeled **Finish.**                                                                                                                   | ![](../.gitbook/assets/finish.png)                       |

## 4. Add field definitions to your data

| Step                                                                                                                                                                                                                                  | Screenshot                                            |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| <p>After completing your data and non-data resource uploads, you will be taken to a private view of your dataset. <br><br>You will see the dataset denoted as Private.</p>                                                            | ![](../.gitbook/assets/private\_dataset\_view.png)    |
| Click the button labeled **Manage** in the upper right                                                                                                                                                                                | ![](../.gitbook/assets/manage.png)                    |
| Click the **Resources** tab                                                                                                                                                                                                           | ![](../.gitbook/assets/resources\_tab.png)            |
| Click on a data resource (e.g. CSV) to which you want to add a data dictionary                                                                                                                                                        | ![](../.gitbook/assets/click\_resource\_to\_edit.png) |
| Click on the **Data Dictionary** tab                                                                                                                                                                                                  | ![](../.gitbook/assets/data\_dictionary.png)          |
| <p></p><p>For each data field, copy information over from the Metadata Template workbook in the Data Dictionary Template sheet:</p><ul><li>Copy Field Label over to Label</li><li>Copy Field Definition over to Description</li></ul> | ![](../.gitbook/assets/add\_data\_definition.png)     |
| Click the button labeled **Save** at the bottom                                                                                                                                                                                       | ![](../.gitbook/assets/save\_data\_dictionary.png)    |
| <p>If you have multiple data files, repeat for each data file by clicking on the button labeled All resources at the top. <br><br>Then select the next file to which you'd like to add data definitions.</p>                          | ![](../.gitbook/assets/all\_resources.png)            |

## 5. Review your dataset **metadata**

| Step                                                                                                                               | Screenshot                                        |
| ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| From the page listing all of your resources, click **View Dataset** in the upper right                                             | ![](../.gitbook/assets/all\_resources\_list.png)  |
| Review your dataset description for human readability and grammar                                                                  | ![](../.gitbook/assets/description.png)           |
| Check that your license is specified as Other (Public Domain) at the bottom of the left-most content                               | ![](../.gitbook/assets/license\_icon.png)         |
| Check the accuracy of the other metadata in the Additional Info table at the bottom                                                | ![](../.gitbook/assets/additional\_info.png)      |
| If you catch any errors or omissions, click **Manage** in the upper right, which will take you back to the form entry for metadata | ![](<../.gitbook/assets/manage (2).png>)          |
| Make changes in the metadata form and click **Update Dataset** at the bottom                                                       | ![](<../.gitbook/assets/update\_dataset (1).png>) |

## 6. Review your resources

| Step                                                                                                        | Screenshot                                       |
| ----------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| From your private dataset page, scroll down to the section labeled **Data and Resources**                   | ![](../.gitbook/assets/data\_and\_resources.png) |
| Click on each resource, which will take you to a preview                                                    | ![](../.gitbook/assets/resource\_preview.png)    |
| If you find any errors or omissions or need to re-upload your resource, click **Manage** in the upper right | ![](<../.gitbook/assets/manage (3).png>)         |
| Go back to the private dataset page and continue to check each resource until done                          |                                                  |

## 7. Share the private dataset with your team for review and approval

{% hint style="warning" %}
**All reviewers must have accounts on data.ca.gov to review the private dataset**

If you haven't checked, make sure all reviewers have access to publishing within your organization on the portal. If you need to request access, [contact the open data team](mailto:opendata@state.ca.gov?subject=\[publisher\_handbook]%20Requesting%20publisher%20permissions%20on%20data.ca.gov).
{% endhint %}

1. You can copy the link to your private dataset and send to the reviewers and work with your Data Coordinator on final publishing approval
2. Reviewers must log in with their accounts to see the private dataset

## 8. Make public after approval

| Step                                                                                                 | Screenshot                                        |
| ---------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| After receiving publishing approval, [login to the open data portal](https://data.ca.gov/user/login) | ![](../.gitbook/assets/login.png)                 |
| Click on the **My Datasets** tab                                                                     | ![](../.gitbook/assets/my\_datasets.png)          |
| Click on the dataset in your list you want to make public                                            | ![](../.gitbook/assets/mydataset\_list.png)       |
| Click **Manage** in the upper right                                                                  | ![](<../.gitbook/assets/manage (1).png>)          |
| Set **Visibility** to **Public**                                                                     | ![](../.gitbook/assets/visibility\_public.png)    |
| Click **Update Dataset** at the bottom of the page                                                   | ![](<../.gitbook/assets/update\_dataset (1).png>) |
| Your dataset is now public                                                                           |                                                   |
